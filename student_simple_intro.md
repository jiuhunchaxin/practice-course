# 【极简导读】我们到底要干什么？

如果用一句话来概括我们的任务：**我们要写一个系统，来计算全国物价到底涨了多少（CPI计算器）。**

---

### 1. 为什么这个问题需要“大数据”？
假设你想知道物价涨没涨，如果只看**猪肉、苹果、衣服**这 3 样东西，你用 Excel 甚至用计算器 1 分钟就能算出来。

**但是真实的国家级场景是：**
每天，全国 50 个城市，都在采集上万种商品的价格。
1 个月下来，这会产生 **1500万条** 价格数据。
如果你用普通的 Python 或 Excel 强行去算，你的电脑会直接卡死、内存爆炸（OOM）。**这就是为什么我们需要大数据架构。**

---

### 2. 作为学生，你需要做的三个极简步骤：

#### 第一步：造数据 (无米之炊怎么做饭？)
因为我们拿不到国家统计局的真实内部数据，所以第一步，我们要写一小段 Python 脚本，像个“造假机器”一样，**随机生成几千万条全国各地的假物价数据**。

#### 第二步：在本地做“微缩版”数学题 (TDD 测试)
在处理那几千万条数据之前，我们要先用 **3 个商品**在自己的电脑上算一遍。
确保你的公式是对的，这叫做“基准测试”。（只要 3 个商品算对了，3000万个商品也一定是这套逻辑）

#### 第三步：把算力丢给云端 (体验大数据的魔法)
重点来了！由于你的电脑算不动千万级数据，所以：
1. 我们把生成好的巨量数据，统统扔到云盘上（阿里云 OSS）。
2. 我们不下载数据，而是写**一行霸气的 SQL 代码**。
3. 借助云端的超级计算引擎（ClickHouse），远程让它在 1 秒钟之内，把几千万条物价数据直接加权合并，算出最终的 CPI 结果。

---

### 3. 我们需要用到哪些工具？（兵马未动，粮草先行）

在这个项目中，你不需要在电脑上安装沉重复杂的大数据软件，我们主要靠 **“本地写代码 + 云端做计算”** 的方式：

*   **💻 你的本地电脑（负责写逻辑和测试）：**
    *   **Python 3**：写代码的基础语言。
    *   **Pandas & Numpy**：用来在第一步“造”出海量假数据的库。
    *   **Pytest**：用来做第二步“微缩版数学题”的测试工具。
    *   **Clickhouse-connect**：Python 的一个“遥控器”插件，让你能通过代码指挥云端的数据库。

*   **☁️ 云端服务（负责承担千万级数据的暴击，由老师提供/配置）：**
    *   **阿里云 OSS (对象存储)**：相当于一个无限容量的“云盘”，负责存放那几千万条历史物价记录。
    *   **ClickHouse 数据库**：超级无敌快的“云端大脑”，负责接收你的 SQL 并在 1 秒内算完海量数据。你无需安装它，只需要用 Python 连上它即可。

*   **⚙️ 自动化管家（负责让一切自动跑起来）：**
    *   **GitHub / GitHub Actions**：写完代码传上去后，它可以充当一个免费的机器人，每个月1号半夜自动帮你跑一遍代码算出最新物价，不用人工盯盘。

---

### 4. 最终你能获得什么？（项目成果与收获）

完成这个项目后，你将不再仅仅是只会写单纯算法的新手，你将拥有：

1. **一个工程级的代码库**：符合企业级规范的模块化架构，告别把所有代码塞进一个文件的学生写法。
2. **“大数据思维”初体验**：彻底理解“计算下推”、“列式存储”等只有在处理千万级数据时才会碰到的真实痛点与解法。
3. **一个杀手级的简历项目**：你可以自豪地在简历上写下：“独立设计并实现基于列式数据库的高频物价指数系统，支持千万级明细数据的秒级实时聚合。”

---

### 💡 核心总结
你在这个项目里，不是在死记硬背复杂的数学公式，而是在体验一种解决问题的思路：**当数据量大到单机无法承受时，如何借助“分离存储”和“云端计算”来秒级算出结果。**
